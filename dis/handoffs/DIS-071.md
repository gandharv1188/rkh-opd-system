# DIS-071 — Shared OcrPort contract test suite

**Status:** Done
**Branch:** `feat/dev-c-wiring-contracts`
**Wave:** 3a (Epic C)
**TDD ref:** §9.1
**Files:** `dis/tests/integration/ocr-port-contract.test.ts`

## 1. What

Parameterized contract suite `runOcrPortContractSuite(factory)` that every
`OcrPort` implementation re-runs. Codifies the CS-2 verbatim-raw-response
invariant plus the `OcrResult` shape at the port boundary so later adapters
(claude-vision, on-prem Chandra) get a free correctness fence.

Applied immediately against two implementations:

- `FakeOcr` (DIS-012) — canned result.
- `DatalabChandraAdapter` — injected mock `fetch` + fake `SecretsPort`,
  no network, loads `tests/fixtures/datalab/convert-response.json` as the
  expected rawResponse body.

## 2. Contract assertions

Per `TDD §9.1` and `adapters.md §Ground rules`:

1. Returns `OcrResult` with the documented shape — `provider`,
   `providerVersion`, `pageCount` (≥ 0), `latencyMs` (non-negative finite),
   `rawResponse` present.
2. `rawResponse` is byte-identical to the provider's reply (CS-2 — audit /
   reprocessing must be reproducible).
3. `latencyMs` is always populated, even on fake adapters.
4. Failure paths throw a named Error subclass (not a bare `Error` / string) so
   callers can branch on `err.name` or `instanceof`.

## 3. Decisions

- **File location.** Brief directs
  `dis/tests/integration/ocr-port-contract.test.ts`; that's what I wrote. The
  backlog (DIS-071 §Files allowed) calls for
  `dis/tests/contracts/{ocr-port.contract.ts,ocr-port.runner.test.ts}` —
  diverged from the brief on purpose because (a) the vitest config only picks
  up `tests/**/*.test.ts`, so a `.contract.ts` file would not run unless
  imported, and (b) keeping the suite + applications together reduces
  cross-file coupling for a single-ticket deliverable. If the team prefers the
  split naming for consistency with the other contract tickets (DIS-072..078),
  the file can be renamed / split without functional change — the exported
  `runOcrPortContractSuite` is the surface.
- **Factory shape.** `factory.happy()` / `factory.failing()` are called per
  test (not once per suite) so every case gets a fresh adapter — avoids
  cross-test state in the scripted-fetch queue.
- **Error-class assertion (rule 4).** Deliberately weak (`name !== 'Error'`
  + length > 0) — tight enough to catch "threw a string" or "threw `new
  Error()`", loose enough that each adapter can own its own error hierarchy
  without the contract naming specific classes. The adapter's own unit tests
  (`tests/unit/adapters/datalab-chandra.test.ts`) continue to pin the exact
  `OcrProviderError` / `OcrProviderTimeoutError` / `OcrProviderRateLimitedError`
  classes.
- **`unknown` rawResponse at the port.** The contract compares `rawResponse`
  with `toEqual` against an `expectedRawResponse` from the factory — callers
  do not introspect it directly, consistent with the port's
  `rawResponse: unknown` typing.
