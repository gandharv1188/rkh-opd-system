# DIS-093 — POST /extractions/:id/reject

## 1. Scope
Thin HTTP wrapper over `IngestionOrchestrator.reject()`. Validates body
shape, extracts actor from body or `X-Operator-Id` header, calls the
orchestrator, and returns a minimal status envelope. Domain errors
propagate to the DIS-101 global `errorHandler` for canonical mapping.

## 2. Files touched
- `dis/src/http/routes/extractions-reject.ts` (new) — route registrar.
- `dis/tests/http/extractions-reject.test.ts` (new) — 6 vitest cases.
- `dis/handoffs/DIS-093.md` (this file).

## 3. Route contract
- `POST /extractions/:id/reject`
- Request body (JSON):
  - `expected_version` (number, required)
  - `reason_code` (string, required, non-empty after trim)
  - `actor` (string, optional; falls back to `X-Operator-Id` header)
- 200 response body: `{ extraction_id, status, version, correlation_id }`
- 400 codes: `MISSING_EXPECTED_VERSION`, `MISSING_REASON`, `MISSING_ACTOR`
- 409: `VERSION_CONFLICT` (via DIS-101 handler from `VersionConflictError`)
- 404: `EXTRACTION_NOT_FOUND` (via DIS-101 handler)

## 4. Validation order
1. Body present and `expected_version` is a `number` → else 400 MISSING_EXPECTED_VERSION.
2. `reason_code` present + non-empty after `.trim()` → else 400 MISSING_REASON.
3. `actor` present in body OR `X-Operator-Id` header → else 400 MISSING_ACTOR.

## 5. Orchestrator wiring
Passes `{ id, expectedVersion, actor, reasonCode }` (matches
`RejectInput` in `dis/src/core/orchestrator.ts:104`). The orchestrator's
optimistic lock + state-machine path is re-used unchanged.

## 6. Error propagation
Route never catches domain errors. `VersionConflictError` and
`ExtractionNotFoundError` thrown by `orchestrator.reject()` bubble up to
`errorHandler()` (DIS-101) which maps them to 409 VERSION_CONFLICT
(non-retryable, details `{current_version, current_status}`) and 404
EXTRACTION_NOT_FOUND respectively.

## 7. Correlation ID
Reads from `c.get('correlationId')` set by the DIS-008 correlation
middleware. `AppVariables` typing imported from `http/server.ts` via a
local `typed` cast of the plain `Hono` app argument, preserving the
registrar's caller-friendly signature while enabling typed context reads.

## 8. Tests (6 cases, all green)
- happy path → 200 + body shape + exact orchestrator input echoed back.
- **"requires non-empty reason"** (VERIFY-2 anchor) — three sub-assertions:
  empty string, missing key, whitespace-only all produce 400 MISSING_REASON.
- version conflict → 409 VERSION_CONFLICT via DIS-101 handler, retryable=false,
  details carry `current_version` + `current_status`.
- missing `expected_version` → 400 MISSING_EXPECTED_VERSION.
- missing actor (no body actor, no header) → 400 MISSING_ACTOR.
- actor fallback via `X-Operator-Id` header → 200, orchestrator sees header value.

## 11. Risks / notes
- `actor` fallback order is body first, header second (matches brief).
  If both are present, body wins; this is deliberate since callers who
  explicitly set it are more specific than the shared proxy header.
- `reason_code` is trimmed before emptiness check to reject whitespace-
  only submissions. The trimmed value is what flows to the orchestrator
  as `reasonCode`, so downstream audit records carry a clean string.
- Junction-linked `dis/node_modules` → main repo's `dis/node_modules`
  was needed to run vitest in this worktree. Created via PowerShell
  `New-Item -ItemType Junction`. Not committed (inside `node_modules`).
