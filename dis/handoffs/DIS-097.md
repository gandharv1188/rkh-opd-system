# DIS-097 — POST /internal/process-job

## Scope
Internal endpoint invoked by the pg_cron worker. Runs
`orchestrator.process(jobId)` and refuses external callers without the
shared worker token.

## Files delivered
- `dis/src/http/routes/process-job.ts` — route handler.
- `dis/tests/http/process-job.test.ts` — 5 cases (incl. VERIFY-2/3 exact names).

## Design notes

### Constant-time token comparison
`X-Worker-Token` is compared against `deps.workerToken` via
`crypto.timingSafeEqual`. Timing leakage on the worker token would
let an attacker probe the secret byte-by-byte via HTTP latency — the
classic authentication side channel — so a constant-time compare is
mandatory even though the token is server-to-server.

The helper handles the two `timingSafeEqual` gotchas:

- Inputs must be equal-length `Buffer`s; mismatched lengths throw.
  Length is not itself secret, so we short-circuit `false` before
  building Buffers (avoids the throw; tiny, fixed time cost).
- Any other throw (encoding edge cases) is caught and returns `false`
  rather than bubbling as 500.

Empty `deps.workerToken` is rejected up front — a mis-wired server
must not silently accept anything.

### Token source
The token is **injected via `ProcessJobRouteDeps.workerToken`**, not
read from `process.env` in this module. Env wiring belongs in the
composition root (the `createServer` caller), keeping the route
pure and directly testable.

### Error propagation
Orchestrator throws (pipeline failures, unexpected errors) propagate
to the DIS-101 global error handler — the handler itself never
catches them. Verified by the "orchestrator throws → 5xx" test
which asserts the DIS-101 `INTERNAL` envelope code.

### Request body
Minimal: `{ extraction_id: string }`. Missing/malformed JSON →
`400 MISSING_EXTRACTION_ID`. No other fields are accepted (forward
extension is cheap when needed).

### Response shape
Matches the sibling routes (`ingest`, `extractions-approve`,
`extractions-reject`, `extractions-retry`):
```
{ extraction_id, status, version, correlation_id }
```
`correlation_id` comes from the correlation-id middleware's context
variable.
