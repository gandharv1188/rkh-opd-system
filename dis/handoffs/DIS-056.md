## 1. What was built

- `dis/src/adapters/queue/pg-cron.ts` — `PgCronAdapter` implementing `QueuePort`.
  - `enqueue(topic, payload, opts?)` → `INSERT INTO dis_jobs (topic, payload, visible_at) VALUES ($1, $2::jsonb, $3) RETURNING id` through the injected `DatabasePort.query`.
  - `opts.delaySec` computes `visible_at = new Date(now() + delaySec * 1000)`; `now` is seam-injectable for deterministic tests.
  - Default `visible_at = new Date(now())` when delay omitted.
  - `safeStringify` uses a `JSON.stringify` replacer that throws on `bigint` / `function` so non-serialisable payloads are rejected at the adapter boundary per the port contract.
  - `startConsumer()` is a no-op — pg_cron + pg_net dispatch HTTP calls to `/internal/process-job` (Epic D / DIS-097); there is nothing for the in-process adapter to do.

No cross-adapter imports (depends only on `DatabasePort`). No new npm deps.
