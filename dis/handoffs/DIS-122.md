## CS-4 invariant

Approve is BLOCKED when `duplicateOfId !== null` AND operator has NOT overridden. Downstream (approve button) should call `isApproveBlocked(...)` to gate the action. Override requires a non-empty reason; the banner calls `onOverride(reason)` to bubble the audit trail up.

UI states:

1. **No duplicate** (`duplicateOfId === null`) → renders nothing.
2. **Duplicate, not overridden** → red `role="alert"` banner with reason input + Override button (disabled when reason blank) + Cancel.
3. **Duplicate, overridden** → amber `role="status"` banner confirming the recorded reason; downstream approve enabled.

## Downstream consumer hint

