## Design Decisions
- **Zero new dependencies.** Brief prohibited `npm install`; `@opentelemetry/api` is not in `dis/package.json`. Implemented a minimal in-house tracer that mirrors the shape of OTel's `withSpan(name, attrs, fn)` so call sites can be lifted to the real SDK later without refactoring.
- **API surface (stable):** `getTracer()`, `setTracer(t)`, `createInMemoryTracer()`, `withSpan(name, attrs, fn)`. These are the only symbols adapters should depend on.
- **Attributes** are a flat `Record<string, string|number|boolean>` (OTel-compatible primitive attribute types). Copied into the span record so mutation after call does not affect the stored span.
- **Error semantics.** On thrown promise: span is recorded with `status: 'error'` + `errorMessage`, and the error is re-thrown — call sites do not need try/catch around `withSpan`.
- **Default tracer** is an in-memory singleton so POC code can call `getTracer().withSpan(...)` without wiring. In staging/prod, `setTracer(realOtelTracer)` swaps the implementation in one place (composition root).

## Adapter usage pattern
```ts
import { getTracer } from '../observability/tracing.js';
const tracer = getTracer();
await tracer.withSpan('adapter.ocr.extract', { provider: 'datalab' }, async () => {
  return await callProvider();
});
```

Naming convention: `adapter.<port>.<method>` (e.g., `adapter.ocr.extract`, `adapter.structuring.structure`).
