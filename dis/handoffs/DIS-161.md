# DIS-161 — PII redactor (CS-8 batched sign-off)

## Scope
Programmatic deep-redact function for trace attributes and ad-hoc payloads. Complements DIS-146 pino-level redaction: DIS-146 scrubs log records on serialize; DIS-161 is usable anywhere (span attributes, enqueue payloads, error metadata).

## Deliverables
- `dis/src/observability/pii-redactor.ts` — `redact<T>(input: T): T` + `__PII_KEYS` test helper.
- `dis/tests/observability/pii-redactor.test.ts` — 4 tests.

## Behavior
- Deep clone; PII values replaced with `'[REDACTED]'`.
- Case-insensitive key match via lowercase lookup.
- PII key set: `patient_name`, `patientname`, `name`, `dob`, `date_of_birth`, `birth_date`, `uhid`, `phone`, `phone_number`, `mobile`, `email`, `email_address`.
- Arrays, primitives, `null`, `undefined` handled safely.

## CS-8 Integration Notes
- Call `redact()` before attaching payloads to OpenTelemetry span attributes.
- Call `redact()` before enqueueing any payload that flows into trace exporters or external observability sinks.
- Pair with DIS-146 pino redaction: pino covers structured logs; this covers everything else.
