## 3. Decisions taken

### D-1: Spy pattern instead of injecting AuditLogger into orchestrator

**Context:** Current `IngestionOrchestrator` does not accept an `AuditLogger` dependency and does not emit audit events natively. The brief forbids src/ writes.
**Decision:** Wrap `logger.write` with a capturing closure in-test, and emit events manually alongside each orchestrator call. Uses the real `AuditLogger.write()` implementation on a `FakeDatabaseAdapter` ŌĆö the `ocr_audit_log` insert path is exercised for every event.
**Reason:** Exercises both the AuditLogger contract and the per-transition coverage requirement without preempting the src wiring ticket. When a future ticket wires `AuditLogger` into the orchestrator, this test's assertions carry over with minimal change ŌĆö the spy becomes unnecessary.

### D-2: Use the OCR-scan pipeline path

**Context:** The native-text path produces only 2 transitions (uploaded ŌåÆ structuring ŌåÆ ready_for_review); OCR-scan produces 4 (uploaded ŌåÆ preprocessing ŌåÆ ocr ŌåÆ structuring ŌåÆ ready_for_review), which plus ingest (ŌåÆ uploaded) and approve (ŌåÆ verified) gives us 6 events ŌĆö safely above the Ōēź 5 threshold.
**Decision:** Script `FakeFileRouterAdapter` on `''` with `{ kind: 'ocr_scan' }`.
**Reason:** More transitions ŌåÆ richer CS-3 coverage. Keeps the test within the integration brief's "fakes only" guardrail.

### D-3: Validate transitions via the pure `transition()` function

**Context:** A weaker test would just assert event count. Brief calls for "every orchestrator method that changes state must emit an audit-log event" ŌĆö implies coverage of valid state-machine edges.
**Decision:** For every captured `state_transition` event, reconstruct the driving `Event` and feed `(from, event)` back through `transition()`, asserting the result equals the captured `toState`.
**Reason:** Catches regressions where an audit event could be emitted for a transition the state-machine would reject. CS-1 (state-machine total function) and CS-3 (audit completeness) are mutually reinforcing ŌĆö this test checks both jointly.
