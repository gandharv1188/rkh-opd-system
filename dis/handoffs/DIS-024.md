## 1. What was built

- `dis/src/core/audit-log.ts` — `AuditLogger` class wrapping a `DatabasePort`. Exposes `write(event)` and `writeMany(events)`. Deliberately has no `update` or `delete` method — immutability is proven at the type level. `AuditLogImmutableError` typed error (with `code='AUDIT_LOG_IMMUTABLE'` and `attemptedOperation` discriminator) is exported so adapters that surface DB-trigger violations can translate them into a domain error.
- `dis/tests/unit/audit-log.test.ts` — 7 unit tests using a fake `DatabasePort` that records inserts. Covers: insert row, required fields carried, transaction used, bulk-write order preserved, update/delete absence at runtime, immutable-error shape for update, immutable-error shape for delete.
- SQL used is a single parameterised `insert into ocr_audit_log (...)` matching the schema in `03_data/data_model.md`. `before_value` / `after_value` are JSON-stringified when present (jsonb columns).

Layering: `audit-log.ts` imports `DatabasePort` from `../ports/database.js` — no adapter imports (port-validator clean).

## 2. Acceptance criteria status
