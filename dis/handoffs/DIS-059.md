## 3. Decisions taken during implementation

### D-1: Injectable clock via `NativePdfTextAdapterOptions.now`

**Context:** latency is computed with `Date.now()`; tests that want deterministic latency should be able to stub it.
**Options considered:** (a) always use `Date.now()`, (b) accept a `now?` option, (c) mock `Date.now` globally in tests.
**Choice:** (b) — same pattern used by sibling adapters; zero runtime cost when unused; keeps production call-sites trivial (`new NativePdfTextAdapter()`).
**Impact:** the happy-path test asserts `latencyMs >= 0` without stubbing, which is sufficient for the contract.

### D-2: Reference-equality not strictly asserted for rawResponse

**Context:** the ticket offered either reference-equality (via `vi.mock`) or structural equality for the CS-2 check.
**Options considered:** spy on `extractNativeText` and assert `===`, vs. assert `result.rawResponse.pages` deep-equals and reconstitutes identical markdown.
**Choice:** structural — simpler, no mock plumbing, still proves no lossy projection. The adapter literally assigns `rawResponse: result`, so reference-equality would be tautological given the source under test.
**Impact:** test remains readable and independent of vitest mocking surface; covers CS-2 intent.
