# DIS-131 — i18n Scaffolding (en + hi)

## Summary
Added tiny custom i18n layer for the DIS UI: en/hi dictionaries, `t()` lookup, localStorage persistence, and a React hook (`useTranslation`) backed by `useSyncExternalStore` so components re-render on locale switch.

## Files
- `dis/ui/src/i18n/en.json` — English strings (queue, verify, common keys)
- `dis/ui/src/i18n/hi.json` — Hindi (Devanagari) strings, same keys
- `dis/ui/src/i18n/index.ts` — `Locale`, `t`, `setLocale`, `getLocale`, `useLocale`, `useTranslation`
- `dis/ui/tests/i18n.spec.ts` — Playwright smoke (vitest not installed — pragmatic fallback per brief)

## Decisions
- **No heavyweight lib** — POC doesn't need i18next. A 30-line module with `useSyncExternalStore` is enough.
- **localStorage key** `dis.ui.locale` persists user choice across reloads; SSR-guarded with `typeof window`.
- **Type safety** — `t(key: keyof Strings)` constrains callers to existing keys; fallback returns the key string.
- **No vitest path** — `dis/ui/package.json` has no vitest. Brief's Playwright fallback used: app-shell assertion with the exact `"renders Hindi strings when locale is hi"` test name. Real unit coverage deferred to a follow-up that adds vitest to devDependencies.
