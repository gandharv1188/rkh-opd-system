# DIS-134 — Skeleton loading states

## Summary
Added reusable `Skeleton` component plus `QueueSkeleton` and `VerifySkeleton` presets for Queue and Verify pages while data loads. Keyframes injected once via a guarded `<style data-skeleton>` tag.

## Files
- `dis/ui/src/components/Skeleton.tsx` (new) — `Skeleton`, `QueueSkeleton`, `VerifySkeleton`.
- `dis/ui/tests/skeleton.spec.ts` (new) — Playwright fallback (vitest/DOM harness not wired in this workspace).

## Component contract
- `SkeletonProps`: `width?` (string | number, default `100%`), `height?` (string | number, default `1rem`), `variant?` (`'text' | 'rect' | 'circle'`, default `'text'`).
- Renders a `<div role="status" aria-label="Loading" data-testid="skeleton">` with pulse animation.
- `QueueSkeleton` — 5 stacked 2rem rows (vertical flex, 0.5rem gap), `data-testid="queue-skeleton"`.
- `VerifySkeleton` — 2-column grid: left 80vh rect (document pane), right 8 x 2.5rem rows (field list), `data-testid="verify-skeleton"`.

## Integration note (follow-up, not in this ticket)
Queue and Verify pages should render these presets while their initial data fetch is pending, then swap them for real content once loaded. Not wired here to keep blast radius small (files-allowed list is strict).
