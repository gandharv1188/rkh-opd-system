## 3. Decisions taken during implementation

### D-1: Inline frontmatter parser rather than a dependency

**Context:** ticket scope is a single `.md` file loaded once at boot; pulling `gray-matter` or `js-yaml` into deps is overkill.
**Decision:** two regex reads — a frontmatter block matcher and a `version:` line finder — raise clear errors if either is missing.
**Revisit if:** we add more structured frontmatter (e.g., model name, token budget) that would justify a real YAML parser.

### D-2: Content hash covers the full file, not just the body

**Context:** the version line itself is content — changing it without editing the body should still produce a different stamp for audit.
**Decision:** `sha256(TEXT)` over the whole file including frontmatter.
**Revisit if:** a future migration extracts frontmatter to a separate manifest.

### D-3: Module-level load with `Object.freeze`

**Context:** the loader's contract is "load once at startup, return the same object for the life of the process". Consumers stamp `contentHash` onto every structuring call, so mutation would be a footgun.
**Decision:** read + parse + hash at module-load time; freeze the result; `getStructuringPrompt()` returns the same reference every call.
**Revisit if:** tests need to swap the prompt at runtime (none do today).

### D-4: node:crypto directly, not DIS-027's `sha256` helper

**Context:** DIS-027 (content-hash) is on a sibling branch and unavailable here per the brief.
**Decision:** `createHash('sha256').update(text,'utf8').digest('hex')`.
**Revisit if:** DIS-027 merges into main and cross-import is desired — trivial swap.

## 4. What was deliberately NOT done

- No wiring into the Claude structuring adapter (DIS-051 owns that).
- No PHI scrubbing — prompt is PHI-agnostic by design.
- No runtime mechanism to hot-reload the prompt (boot-time read is sufficient for auditability; prompt rev = process rev).
- No consolidation with the pre-existing `dis/src/prompts/structuring.md` (outside this ticket's files_allowed; that path is left untouched).
