## 1. What was built

- `dis/src/adapters/structuring/claude-haiku.ts` — `ClaudeHaikuAdapter` class implementing `StructuringPort`.
  - Loads prompt + schema synchronously at construction (cold path, keeps wiring simple).
  - Extracts `version` from YAML frontmatter via regex (no `yaml` dep).
  - Anthropic client injected via `AnthropicClientFactory(apiKey) => AnthropicLike` so tests never hit the SDK or network.
  - Validates the model reply with a lean hand-rolled required-keys check (per task constraint: no Ajv).
  - On first parse/validation failure, retries **once** with a stricter user cue ("SINGLE valid JSON object … no prose, no markdown fences").
  - On second failure, throws `StructuringSchemaInvalidError` (exported, carries `attempts` + `lastRaw` for diagnostics).
  - Cost accounting at Haiku price points (83 µINR in / 416 µINR out per token) — placeholder until pricing ticket.
  - Returns `StructuringResult` with `promptVersion` stamped from frontmatter, raw response preserved verbatim (CS-2 flavour).
- `dis/src/prompts/structuring.md` — YAML frontmatter `version: 1` + 3 few-shot examples (lab report, prescription, vaccination card) covering the 4 array fields in the schema.
- `dis/src/schemas/clinical-extraction.v1.json` — JSON Schema draft-07, required `document_type`, `summary`, `labs`, `medications`, `diagnoses`, `vaccinations`. `additionalProperties: true` (forward-compat — strict mode can land when Ajv is introduced).

Validation layering: adapter → port (`StructuringPort`) → no cross-adapter imports (hex-arch clean).
