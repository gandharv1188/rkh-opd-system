## 3. Decisions taken during implementation

### D-1: Chose Hono over Express for HTTP layer

**Context:** DIS coding_standards §1 mandates portable HTTP framework.
**Options considered:** Hono, Express, Fastify.
**Decision:** Hono + `@hono/node-server`.
**Reason:** Edge-portable (matches existing Supabase Edge Functions stack); small surface; first-class Web Fetch API test ergonomics (`app.fetch()` needs no listening port).
**Revisit if:** we need Express-specific middleware ecosystem.

### D-2: Correlation-id generated as UUIDv4 via `crypto.randomUUID()`

**Context:** middleware must mint id when inbound header absent.
**Decision:** Node 20 built-in `crypto.randomUUID()`; no extra dep.
**Revisit if:** we need ULID/KSUID for sortable ids.

### D-3: No `package.json` edit

**Context:** DIS-001 owns `dis/package.json`.
**Decision:** Declare deps in `DEPS_REQUIRED.md` only; integrator merges.
**Revisit if:** DIS-001 lands and we have a standalone merge wave.
