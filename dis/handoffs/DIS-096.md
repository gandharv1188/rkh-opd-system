## 3. Decisions taken during implementation

### D-1: Named pass-through to match actual StoragePort API (not brief)

**Context:** Brief prescribed `storage.createSignedUploadUrl({ key, contentType, expiresInSeconds })` returning `{ url, key, expiresAt }`. Actual port in `dis/src/ports/storage.ts` is `getSignedUploadUrl({ key, contentType, expiresSec, maxSizeBytes })` returning `{ kind: 'signed-upload', url, fields? }`.
**Decision:** Call real method with real field names; compute `target_path` from the key I generate and `expires_at` from `Date.now() + ttl*1000` since the result carries neither.
**Reason:** Adjusting to reality beats synthesising a shim; team-lead approved explicitly.
**Revisit if:** a future adapter supplies a server-side `expiresAt` (e.g. S3 presigner) — switch to that value.

### D-2: Require `max_size_bytes` with 25 MiB default

**Context:** Port mandates `maxSizeBytes`, brief didn't mention it.
**Decision:** Accept optional `max_size_bytes` in body; default to 25 MiB (`25 * 1024 * 1024`).
**Reason:** Matches typical clinical-document size ceiling; keeps the route self-sufficient without a config plumb.
**Revisit if:** product decides on a different cap or wants per-role caps.

### D-3: Key pattern `uploads/<patient_id>/<epoch-ms>-<filename>`

**Context:** Brief suggested this shape.
**Decision:** Kept verbatim; no sanitisation of `filename` yet.
**Revisit:** path traversal / unicode-homoglyph hardening is a follow-up (see §5).

## 4. What was deliberately NOT done

- No filename sanitisation (`..`, leading `/`, NUL, zero-width chars). Follow-up ticket recommended.
- No authz / operator scoping — handler trusts the client; kill-switch + rate-limit middleware still gate the route.
- No content-type allowlist — any `content_type` passes through to the port.
- Route not wired into `createServer` (`server.ts`). Intentional: brief restricted edits to the 3 allowed files. Composition wiring is a follow-up (see §5).
- No integration test against a real Supabase signer — the fake port is sufficient for this layer.
