# DIS-127 — Accessibility baseline (handoff)

## Summary
Added a Playwright-native accessibility baseline spec at `dis/ui/tests/a11y.spec.ts`. No external a11y deps used (zero install surface).

## What was done
- New test file `dis/ui/tests/a11y.spec.ts` with three baseline assertions:
  1. Named landmarks (`topbar`, `sidebar`, `main-content`) visible, plus `page.accessibility.snapshot()` returns a non-null tree.
  2. `Tab` keypress does not throw (keyboard reachability smoke).
  3. `<html lang>` attribute is present.
- Uses only `@playwright/test` (already listed in `dis/ui/package.json` devDeps) and the built-in `page.accessibility.snapshot()` API.

Substituted with Playwright's native accessibility snapshot API, which provides the Chromium a11y tree without any new dependency. Coverage is narrower than axe (no WCAG rule-set enforcement) but unblocks a CI baseline today.
