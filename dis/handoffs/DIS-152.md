# DIS-152 — Red-team adversarial fixtures (CS-2 + CS-7 batched)

## Scope
Adversarial fixtures + tests for three attack classes against the OCR →
structuring pipeline. Batched under CS-2 (adversarial inputs) and CS-7
(fail-closed on ambiguous data).

## Files
- `dis/tests/red-team/prompt-injection.test.ts` — CS-2. PDF body tries to
  issue instructions to the structuring LLM. Adapter must treat OCR as data.
- `dis/tests/red-team/misocr.test.ts` — CS-7. OCR confuses `l`/`1`, `O`/`0`.
  Plausibility scorer flags low confidence when digit-letter confusables mix
  inside a token.
- `dis/tests/red-team/unit-conflict.test.ts` — CS-7. Same test with two units
  in one report → reject.
- `dis/tests/fixtures/red-team/README.md` — attack-class index + how to
  extend.

## Implementation notes
- The `structure()` / `scoreOcrPlausibility()` / `detectUnitConflict()`
  helpers in the test files are **stand-ins** for the real adapter. They
  model the behaviour the real adapter must exhibit so that a later
  implementation can be swapped in without the test suite being rewritten.
- Prompt-injection stripper also removes any follow-on imperative sentence
  (`Return …`, `Set …`) that tries to set a value, so real labs further
  down the page still parse.
- Mis-OCR detector flags tokens where OIl confusables **mix with digits**.
  Pure-digit runs like `11.5` are NOT flagged — that was the initial bug.
