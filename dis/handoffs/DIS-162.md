## Summary
Rolling SHA-256 Merkle-style chain over audit-log rows. Each node hashes `prev || "|" || canon(row)`. Tamper detection via `firstDivergenceIndex` (row-level localisation) and `findTamperIndex` (root-only check).

## Files
- `dis/src/observability/audit-integrity.ts` — `AuditRow`, `computeRowHash`, `computeChainHash`, `findTamperIndex`, `firstDivergenceIndex`.
- `dis/tests/observability/audit-integrity.test.ts` — 3 tests (stability, tamper detection, intact chain).

## Deviations from brief
- Exposed `computeRowHash(prev, row)` helper to avoid the dynamic `await import('node:crypto')` pattern in the test. Keeps the test clean and gives storage callers a single-row incremental API.
- `canon()` relies on caller emitting payload keys in consistent order (matches brief). Downstream producers should normalise via a stable stringifier if payloads are user-controlled — noted as a future hardening item.

## CS-5 batched sign-off context
This module sits alongside the other CS-5 observability primitives. Intended usage: persist per-row hash alongside the audit row; periodic batch verifier calls `firstDivergenceIndex` to localise tampering. Root-hash comparison via `computeChainHash` works for quick integrity pings where row-level localisation is not needed.
