## 1. What was built

- `dis/src/adapters/ocr/claude-vision.ts` — `ClaudeVisionAdapter` implementing `OcrPort`.
  - Each page in the OCR input submitted as a single image/jpeg content block (base64 source) + a fixed extraction prompt to the Anthropic Messages API.
  - Per-page replies concatenated (separated by `\n\n`) into the returned `markdown`.
  - `rawResponse` holds `{ pages: <raw reply array> }` — every per-page reply preserved verbatim so CS-2 audit/reprocess can diff.
  - `tokensUsed` aggregated across pages for cost tracking.
  - `providerVersion` / `model` constructor-configurable; defaults `sonnet-4.6` / `claude-sonnet-4-6`.
  - `AnthropicVisionClientFactory(apiKey) => AnthropicVisionLike` seam mirrors DIS-051 — tests inject a fake; no SDK/network required.
  - `defaultFactory` throws with a clear wiring message (DIS-079 owns composition).
  - `stripFences` tolerates models that wrap full output in ` ```markdown ` fences.

No cross-adapter imports (hex-arch clean). No new npm deps.
