# DIS-124 Handoff — Offline-first localStorage edit store

## 1. Summary
Implemented an in-browser, offline-first edit store keyed by `extraction_id`.
Persists in-progress field edits to `window.localStorage` under the
`dis.ui.edit.<id>` key so a tab reload does not lose operator work. Falls
back to an in-process `Map` when `localStorage` is unavailable (SSR, private
mode, or quota errors).

## 4. Public API
```ts
saveEdits(extractionId: string, edits: FieldEdits): void
loadEdits(extractionId: string): FieldEdits | null
clearEdits(extractionId: string): void
__resetForTests(): void  // dual-layer wipe (localStorage prefix + memory Map)
```
`FieldEdits` is a readonly `{ [fieldKey: string]: string }` record.

## 5. Design notes
- `hasStorage()` guards every `window.localStorage` reference with a `try/catch`
  so the module is safe in SSR, disabled-cookie, or private-mode contexts.
- Storage errors (quota exceeded, `SecurityError`, malformed JSON) fall through
  to the in-memory `Map`, keeping the API total.
- Key prefix `dis.ui.edit.` keeps our records out of the way of other features
  that may share `localStorage`.
- `__resetForTests` iterates keys in reverse so `removeItem` during iteration
  stays correct on engines where `length` shifts live.

## 6. Test-runner choice
`dis/ui/package.json` does **not** include `vitest`; only
`@playwright/test` is listed. Per the brief's fallback instruction the spec
uses Playwright's `test` runner, mirroring the existing
`tests/queue.spec.ts` pattern where pure-logic modules are imported and
exercised directly in the Playwright node runner.

In the Playwright node runner there is no `window`, so `hasStorage()`
returns `false` and the store exercises its in-memory fallback path. This is
the correct test surface for CI: the localStorage branch is the thin wrapper
that delegates to `window.localStorage`, whose persistence contract is
browser-defined and not useful to re-test here. The memory fallback is the
only code we actually own, and the spec drives it through the full public
API (roundtrip, reload roundtrip, clear, unknown-id).

no tokens, no secrets. `localStorage` is same-origin by design.
