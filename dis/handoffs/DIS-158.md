# DIS-158 Handoff — Admin Dashboard Skeleton

**Owner:** dev-f-dashboard
**Branch:** feat/dev-f-dashboard
**Status:** Complete (spec only; Grafana JSON out of scope).

## Deliverable

- `dis/document_ingestion_service/10_handoff/dashboard.md` — panel specification with 6 sections:
  1. Queue Depth (gauge)
  2. Throughput (stacked-bar, approved vs rejected per hour)
  3. OCR Latency (time-series p50/p95)
  4. Cost Today (gauge + burn-down vs `DIS_DAILY_BUDGET_INR`)
  5. Error Rate (stacked-bar by stage: preprocess / ocr / structuring)
  6. Operator SLO (p95 verify latency vs DIS-159 target)

Each panel documents: metric source (pointing to DIS-148 Prometheus names), chart type, and alert thresholds (WARN / CRIT).

## Dependencies

- **DIS-148** — metric catalog. Panels reference metrics by canonical Prometheus names (`dis_queue_depth`, `dis_documents_total`, `dis_ocr_duration_seconds_bucket`, `dis_cost_micro_inr_total`, `dis_stage_errors_total`, `dis_verify_latency_seconds_bucket`). If DIS-148 renames any of these, update the dashboard spec in lock-step.
- **DIS-099** — `GET /admin/metrics` endpoint is the single data source. No client-side aggregation beyond trivial sums.
- **DIS-159** — supplies the verify-latency p95 SLO target used by the Operator SLO panel (currently ≤ 4 min during staffed hours).

## Out of scope

- Grafana dashboard JSON.
- Auth/RBAC for the admin dashboard page.
- Alert routing (Alertmanager / pager). Thresholds here are the contract; routing is a separate ticket.
