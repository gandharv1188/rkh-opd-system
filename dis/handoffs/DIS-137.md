# DIS-137 — CSS theme tokens

**Owner:** dev-e-theme
**Branch:** feat/dev-e-theme
**Wave:** 6b-1

## Summary

Added CSS custom properties matching the HMIS prescription pad Royal Blue
(#1a237e) accent, plus spacing/type/elevation/radius scales and dark-mode
overrides, in a single `tokens.css` file. Included a vitest-style spec that
greps the file for the required token definitions.

## Files

- `dis/ui/src/theme/tokens.css` (new) — 60 CSS variables under `:root` plus a
  `prefers-color-scheme: dark` override for bg/text/border tokens.
- `dis/ui/tests/theme.spec.ts` (new) — asserts `--royal-blue: #1a237e`,
  `--dis-accent: var(--royal-blue)`, `--dis-space-4: 1rem`.
- `dis/handoffs/DIS-137.md` (this file).

## Token surface

- **Accent:** `--royal-blue`, `--royal-blue-light`, `--royal-blue-dark`,
  `--dis-accent*`, `--dis-accent-on`.
- **Semantic:** `--dis-bg`, `--dis-bg-muted`, `--dis-text`, `--dis-text-muted`,
  `--dis-border`.
- **Status (DIS-125 aligned):** pending / ready / promoted / rejected each with
  `-bg` + `-fg` pair.
- **Spacing (4px base):** `--dis-space-{1,2,3,4,6,8}`.
- **Type:** `--dis-font-sans`, `--dis-font-devanagari`, five size steps.
- **Elevation:** `--dis-shadow-sm`, `--dis-shadow-md`.
- **Radii:** `--dis-radius-sm`, `--dis-radius-md`.

## Verify

- **VERIFY-1 (vitest):** vitest is not installed in the shared `dis/ui`
  node_modules (only `vite`, `typescript`, playwright are present) and the
  junction prevents node-resolve from finding vite via the worktree path. Skipped
  per brief fallback ("grep-based VERIFY-2 always works").
- **VERIFY-2 (grep):** PASS — `royal-blue` / `--dis-accent` both match in
  `tokens.css` (7 hits on lines 6–12).
- **VERIFY-3 (typecheck):** tokens.css is pure CSS so typecheck is not
  applicable to the artifact. The spec is trivial `readFileSync` + regex with no
  external types needed beyond `vitest`.

## Consumer notes

Import once at app bootstrap alongside `styles.css`:

```ts
import './theme/tokens.css';
```

Subsequent components should reference variables (`color: var(--dis-accent)`)
rather than hard-coding `#1a237e`, so the dark-mode override and future theme
swaps take effect automatically.
