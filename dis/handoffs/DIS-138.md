# DIS-138 — PrintSummary (Printable verification summary)

**Owner:** dev-e-print
**Branch:** feat/dev-e-print
**Status:** Implementation complete

## Scope

Printable A4 verification summary listing extracted fields, confidence, and bbox references for offline audit retention alongside paper charts (NABH §5.2).

## Files delivered

- `dis/ui/src/pages/PrintSummary.tsx` — React component with props `{ extractionId, patientId, fields[], verifiedBy?, verifiedAt? }`. Inline mm-based A4 sizing (210mm × 297mm, 15mm padding) + inline `@page { size: A4; margin: 12mm 10mm }` print media rule. Hospital header in Royal Blue (`#1a237e`), field table with Field/Value/Confidence/Page-BBox columns, footer with NABH retention note.
- `dis/ui/tests/print-summary.spec.ts` — Playwright smoke test, name exactly `"layout renders on A4"`, navigates `/` and asserts topbar. Component-level rendering deferred to harness follow-up (DIS-138-followup) per brief.

## Testids exposed

`print-summary`, `print-header`, `print-fields`, `print-footer`.

Brief prohibits `npm install`. Both the global `npx` cache and auto-install variants fail with `ERR_MODULE_NOT_FOUND` when `playwright.config.ts` imports `@playwright/test`. This is an environment-wide gap affecting all UI worktrees, not DIS-138-specific.
- **VERIFY-3 (typecheck):** Same underlying issue — `tsc` needs the workspace dependencies that are absent. Component is standard JSX/TS with no custom imports; types are all inline interfaces.

The component itself matches the brief's exact specification character-for-character (props interface, JSX structure, testids, inline styles, @page rule) — it is ready to pass harness verification once the UI workspace dependencies are installed by the orchestrator.
