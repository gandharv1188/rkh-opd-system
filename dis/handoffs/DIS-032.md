## 1. What was built

- `dis/src/core/cost-calculator.ts` — exports `calculateCost(usage, rates?) → CostBreakdown`. Pure function, no I/O. Shapes: `Provider = 'haiku' | 'sonnet' | 'datalab' | 'datalab+haiku'`, `RateTable` (per-provider), `UsageInput`, `CostBreakdown` (input/output/ocr costs + total in micro-INR, plus echo of usage counts).
- `dis/tests/unit/cost-calculator.test.ts` — 7 unit tests: haiku / sonnet / datalab / datalab+haiku happy paths, zero-input edge case, env-default fallback when no `rates` passed, and rate-override supersedes env defaults.

Default rates are the ADR-007 placeholders (Haiku 83 / 416, Sonnet 415 / 2080 µINR/token, Datalab 3000 µINR/page). Each placeholder is overridable via env var (`DIS_RATE_*_MICRO_INR`) or via an explicit `rates` argument. Env reads are kept local to this module rather than extending `env.schema.ts` (out of scope — DIS-010 env surface frozen for Wave 2).
