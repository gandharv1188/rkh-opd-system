Title: `# Runbook: Stuck Job (processing state past SLA)`

Six numbered steps (exceeds spec minimum of 5):

1. Detect the stuck-job condition — `queue_depth` gauge + per-job age probe, alert triage.
2. Identify the stuck job id(s) — operational `SELECT` on `ocr_extractions` filtered by `status = 'processing' AND updated_at < NOW() - interval '15 minutes'`.
3. Diagnose the failure mode — three branches: worker down, OCR hung, structuring loop.
4. Retry via `POST /extractions/:id/retry` — operator token, idempotency, verification SELECT.
5. Mark as failed manually if retry is exhausted — guarded `UPDATE` to `status='failed'` with error code `STUCK_MANUAL_FAIL`, escalation threshold.
6. Post-mortem — 24 h SLA, template location, follow-up tickets, runbook update contract.
