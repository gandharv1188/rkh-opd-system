# DIS-160 — GET /health/deep deep probe route

**Owner:** dev-f-health-deep
**Branch:** feat/dev-f-health-deep
**Status:** complete

## Summary
New Hono route `GET /health/deep` probes optional dependencies (db, storage, secrets, ocr) and reports per-component status with latency. Fail-open design: missing deps are simply omitted from the report rather than fabricated. Route registration is opt-in via `registerHealthDeepRoute(app, deps)` — orchestrator wires into `createServer` in a follow-up.

## Files
- `dis/src/http/routes/health-deep.ts` — route + `probe()` helper + `HealthDeepDeps` / `ComponentReport` types
- `dis/tests/http/health-deep.test.ts` — 3 tests (ok / degraded secrets cache miss / 503 on down)

## Behavior
- `probe(component, fn)` wraps any async check with `Date.now()` latency timing and catches errors into `{status: 'down', message}`.
- Storage probe is a placeholder `Promise.resolve()` — real ping is adapter-specific, out of scope here.
- Secrets: calls `secrets.get('DIS_HEALTH_PROBE_NOOP')` with `.catch(() => undefined)` (missing key is not a failure). `secretsCacheMiss=true` short-circuits to `degraded`.
- OCR: mocked `ok` for POC.
- Worst status aggregation: any `down` → 503; any `degraded` → 200; else 200.
- Response body includes `correlation_id` from `AppVariables`.
