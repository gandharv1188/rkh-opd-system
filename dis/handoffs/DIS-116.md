**Owner:** dev-e-queue
**Branch:** feat/dev-e-queue

## Scope delivered

- `dis/ui/src/pages/Queue.tsx` — React page component that lists extraction rows with:
  - Status filter `<select>` (All / ready_for_review / promoted / rejected).
  - Cursor pagination via a `Load more` button shown when `next_cursor` is non-null.
  - Refetch on filter change (effect depends on `statusFilter` via `useCallback`).
  - Error surface (`data-testid="queue-error"`) and empty-state surface (`data-testid="queue-empty"`).
  - Exported pure helper `buildExtractionsUrl(base, statusFilter, cursor)` to keep URL construction testable without React.
- `dis/ui/tests/queue.spec.ts` — Playwright suite (6 tests, all green):
  - Four pure-logic tests for `buildExtractionsUrl` covering no-filter / status-only / cursor-only / both.
  - Two browser tests that exercise the `/extractions` route-mocking strategy against the live app shell: first-page returns 3 items + cursor `ext-3`; cursor page returns `ext-4` + `next_cursor: null`. The second test is named exactly `loads next page on scroll` (required by VERIFY-2).

## Fetch-mocking strategy

Playwright `page.route('**/extractions*', …)` intercepts the request, reads `cursor` from the URL, and returns either the first page or the cursor page. Tests call `fetch('/extractions')` from inside the browser context via `page.evaluate`, which verifies the mock wiring end-to-end without requiring a live backend.
