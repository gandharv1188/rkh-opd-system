# DIS-057 — DefaultFileRouter (FileRouterPort) Handoff

## Summary

Implements TDD §7 file routing decision tree as `DefaultFileRouter` in
`dis/src/adapters/file-router/default.ts`. Satisfies `FileRouterPort` from
`dis/src/ports/file-router.ts`.

## Decision tree (TDD §7)

- `.pdf` → run PDF text extraction; if average chars/page ≥ threshold
  (default `100`) → `{ kind: 'native_text', pageCount }`; else
  `{ kind: 'ocr_scan', pageCount }`.
- `.jpg .jpeg .png .heic .webp .bmp .tiff` → `{ kind: 'ocr_image' }`.
- `.docx .doc` → `{ kind: 'office_word' }`.
- `.xlsx .xls .csv` → `{ kind: 'office_sheet' }`.
- anything else → `{ kind: 'unsupported', reason: 'disallowed_extension' }`.

Classification uses the lower-cased filename extension. `contentType` is
ignored for now (filename ext is the source of truth per TDD §7).

## Configuration

Constructor: `new DefaultFileRouter(options?)` with:

- `nativeTextMinCharsPerPage?: number` — overrides the default `100`. The
  Architect should wire this from `DIS_NATIVE_TEXT_MIN_CHARS_PER_PAGE` env
  at the composition root (wiring layer), not inside the adapter.
- `pdfTextExtractor?: (buf: Buffer) => Promise<{ pages: string[] }>` —
  dependency injection seam used by unit tests. Default uses a lazy
  `import('pdfjs-dist')`.

## Tests

`dis/tests/unit/adapters/file-router.test.ts` covers all TDD §7 branches
plus threshold override (9 cases total). All tests inject a fake
`pdfTextExtractor`, so running them does not require `pdfjs-dist` to
resolve.

## Dependencies

Added to `dis/DEPS_REQUIRED.md`: `pdfjs-dist ^4.7.0`. The Architect must
merge this into `dis/package.json` at integration time per DIS-001 policy.
