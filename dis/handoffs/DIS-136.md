# DIS-136 — Operator Sign-In

## Scope
- `dis/ui/src/auth/SignIn.tsx` — email+password form; POSTs to configurable endpoint (default `/auth/sign-in`); calls `onSignedIn(access_token, operator_id)` on success.
- `dis/ui/tests/sign-in.spec.ts` — Playwright shell tests (`shell renders`, `rejects wrong credentials`).

## Design notes
- **Tokens in memory only.** Access token is handed to the parent via `onSignedIn`. No localStorage / sessionStorage writes for credentials or tokens by design.
- **Error UX.** 401 → "Invalid email or password". Other non-OK → "Sign-in failed". Network/fetch throw → "Network error — try again." Errors are shown in an inline `role="alert"` element (`data-testid="sign-in-error"`).
- **A11y.** `<label htmlFor>` bound to inputs, `autoComplete="email"` / `"current-password"`, submit button disabled while in-flight with a pending label.

## Follow-ups
- **DIS-136-followup: refresh-token flow.** In-memory-only tokens mean a page refresh signs the operator out. A silent refresh (HTTP-only cookie + `/auth/refresh`) is out of scope here and tracked separately.
- Wire `SignIn` into the app shell once the topbar/main-content route is defined.
