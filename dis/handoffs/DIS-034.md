## Summary

New integration test at `dis/tests/integration/state-machine-happy-path.test.ts`
drives `IngestionOrchestrator` end-to-end using DIS-012 script-driven fakes:

```
uploaded  → (routed_native)  → structuring
          → (structured)     → ready_for_review
          → (nurse_approve)  → verified
          → (promoted)       → promoted       [via pure transition()]
```

All transitions routed through `transition()` (CS-1). No real adapters.

## Test cases

1. **Happy path (CS-1 positive):** ingest → process → approve, asserting
   statuses `ready_for_review` → `verified` and that `transition('verified',
   {kind:'promoted'})` returns `'promoted'`. Also asserts exactly two
   persisted status updates in the expected order.
2. **CS-1 negative — premature approve:** calling `orch.approve` while the
   row is still `uploaded` throws `InvalidStateTransitionError`. Row's
   status and version remain unchanged.
3. **CS-1 negative — promote without verify:** `transition('uploaded',
   {kind:'promoted'})` and `transition('ready_for_review', {kind:'promoted'})`
   both throw `InvalidStateTransitionError`.
