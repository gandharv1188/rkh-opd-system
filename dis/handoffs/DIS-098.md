# DIS-098 — Realtime status push (in-process POC publisher)

## Scope
In-process `extraction.status.changed` publisher for the verification UI. Implemented as an EventEmitter-backed `StatusChannel` with a narrow `publish` / `subscribe` port, so the Supabase Realtime (and future AWS SNS) adapter can plug in behind the same interface without churning call sites.

## Files
- `dis/src/http/realtime/status-channel.ts` — `StatusChannel` class + `StatusChangedEvent` + `createStatusChannel()` factory.
- `dis/tests/http/realtime-status.test.ts` — 4 tests (VERIFY-2 name exact: "emits event on state transition").

## What this is / isn't
- **Is**: the abstraction layer used by orchestrator / post-approve hooks for POC. Zero external deps, in-process fan-out only.
- **Isn't**: the production transport. Supabase Realtime adapter should land at `dis/src/adapters/realtime/supabase-realtime.ts` in a follow-up ticket, implementing the same publish/subscribe shape so the composition root can swap it in without touching callers.

## State type alignment
`from_status` / `to_status` use the `State` union from `dis/src/core/state-machine.ts` (`uploaded` | `preprocessing` | `ocr` | `structuring` | `ready_for_review` | `auto_approved` | `verified` | `promoted` | `rejected` | `failed`). Tests use `ready_for_review → verified` as the canonical nurse-approval transition.
