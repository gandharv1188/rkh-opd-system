# DIS-139 — Feature flags hook (handoff)

**Branch:** `feat/dev-e-feature-flags`
**Owner:** dev-e-feature-flags
**Status:** Complete

## Summary
Lightweight feature-flag system for the DIS UI. Module-level store with a
React subscription (`useSyncExternalStore`) so flag changes re-render
consumers, plus an async loader that calls `/admin/flags` and silently
falls back to defaults when the endpoint is unreachable or non-ok.

## Files touched
- `dis/ui/src/flags.ts` (new) — store, `getFlags`, `setFlags`, `loadFlags`, `useFlags`, `useFlag`, `useAutoLoadFlags`.
- `dis/ui/tests/flags.spec.ts` (new) — 5 pure-logic tests under the existing Playwright harness.
- `dis/handoffs/DIS-139.md` (this file).

## Flags + defaults
| Flag | Default | Notes |
| --- | --- | --- |
| `realtime_enabled` | `true` | Supabase realtime subscriptions on/off. |
| `i18n_hindi` | `true` | Hindi copy + Devanagari fonts. |
| `experimental_bbox_overlay` | `false` | OCR bbox overlay on PDF viewer. |
| `print_summary` | `true` | Visit-summary block on printed Rx. |

Defaults are preserved on offline / 5xx / throw. Server partial payloads
merge over defaults; unspecified keys retain prior values.

## Public API
```ts
getFlags(): FeatureFlags
setFlags(partial: Partial<FeatureFlags>): void
loadFlags(endpoint?: string): Promise<void>
useFlags(): FeatureFlags
useFlag<K>(key: K): FeatureFlags[K]
useAutoLoadFlags(endpoint?: string): void
```

## Test runner choice
`vitest` is **not** in `dis/ui` devDependencies — only `@playwright/test`.
Rather than add a dep, the spec is written as a Playwright file but
exercises pure logic (no `page.goto`), so the webServer boots once and the
5 tests run in ~6 ms each.
