# DIS-109 — E2E realtime channel roundtrip

## 1. Ticket

## 3. Approach
Exercise `StatusChannel.publish()` → `subscribe()` in-process. Race the delivery against a 2000ms timer; assert delivery wins, the listener receives exactly one event, and the payload is identical to what was published.

The current publisher is synchronous (`EventEmitter.emit`) — delivery happens on the same tick, so the 2s bound is trivially satisfied today. The race structure exists to keep the bound meaningful when the adapter is later swapped for an async transport (e.g. Supabase Realtime), where latency could actually approach the budget.

## 4. Test spec
`describe('E2E realtime channel roundtrip') > it('event arrives within 2s')` — constructs a `StatusChannel`, subscribes, publishes a fully-populated `StatusChangedEvent` (`from_status='ready_for_review'`, `to_status='promoted'`, `version=2`), and asserts: arrived-in-time === true, `received.length === 1`, `received[0]` deep-equals the published event. Unsubscribes at the end.
