# DIS-165 — Cost Budget Guardrail (CS-12) Handoff

**Status:** CLOSED — CS-12 batched sign-off.
**Owner:** dev-f-cost-guardrail
**Branch:** feat/dev-f-cost-guardrail

## Summary

Introduces a pure, stateless budget-evaluation primitive that refuses new
uploads once the daily cost ledger exceeds `DIS_DAILY_BUDGET_INR`.

## Files

- `dis/src/core/cost-guardrail.ts` — `evaluateBudget`, `assertBudgetAvailable`,
  `BudgetExhaustedError` (code `BUDGET_EXHAUSTED`, retryable).
- `dis/tests/core/cost-guardrail.test.ts` — 3 tests covering under-budget,
  over-budget (the `returns 503 when budget exceeded` case), and exact-budget.

## Contract

- Units are **micro-INR** to match the cost ledger (DIS-146 series).
- `assertBudgetAvailable` throws `BudgetExhaustedError` when
  `consumed >= dailyBudget`. Exact-budget also blocks (fail-closed).
- The HTTP layer (DIS-101 error handler) is responsible for mapping
  `code: BUDGET_EXHAUSTED` to HTTP **503 Service Unavailable** with
  `Retry-After`. This module is transport-agnostic.
