# DIS-094 ‚Äî POST /extractions/:id/retry

## 1. Ticket
Expose `IngestionOrchestrator.retry()` as an HTTP route. The original
extraction is preserved for audit; the new extraction links back via
`parent_extraction_id`. Backlog line 2122. TDD ¬ß4.

## 2. Scope
- New route file `dis/src/http/routes/extractions-retry.ts`.
- New test file `dis/tests/http/extractions-retry.test.ts`.
- No server.ts wiring (reserved for the composition-root ticket that
  mounts all Wave-5b routes together).
- No orchestrator changes ‚Äî contract `retry({ id, actor })` already exists.

## 3. Route contract
- `POST /extractions/:id/retry`
- Actor resolution: JSON body `{ actor?: string }` OR `X-Operator-Id`
  header. Missing ‚Üí `400 MISSING_ACTOR`.
- Idempotency-Key header **not** required: orchestrator applies
  `idempotencyKey: retry:${newId}` internally.
- Success ‚Üí `201 Created` with
  `{ extraction_id, parent_extraction_id, status, version, correlation_id }`.
- Errors flow through the DIS-101 global envelope via thrown
  `HttpError` / `ExtractionNotFoundError`.

## 4. Test coverage (4 cases)
1. Happy path ‚Äî 201 + new `extraction_id`, `parent_extraction_id === :id`.
2. `it("old extraction remains readable after retry")` ‚Äî asserts the
   response body preserves the parent reference (see ¬ß8).
3. Missing actor ‚Üí 400 `MISSING_ACTOR`.
4. `ExtractionNotFoundError` propagates ‚Üí 404 `EXTRACTION_NOT_FOUND`.

Fake orchestrator is a one-method duck (`{ retry }`) cast to
`IngestionOrchestrator` ‚Äî keeps the test suite pure HTTP.

## 5. Implementation notes
- Hono base `Hono` (not `Hono<{ Variables: AppVariables }>`) ‚Äî matches
  the existing `admin-metrics` / `health` route signatures so the
  composition-root ticket can register this alongside the others without
  a generic mismatch. `c.get('correlationId')` therefore needs a
  narrow cast (documented inline).
- Body parse uses `.catch(() => ({}))` so a missing / invalid JSON body
  is treated as "no body" rather than a 500.

## 8. Open decisions / caveats
**VERIFY-2 interpretation.** "Old extraction remains readable after
retry" is really a full-stack invariant (parent row not mutated, still
fetchable by id). At the HTTP-route test boundary the strongest honest
signal is: *the response preserves a reference to the parent id*. The
test asserts `parent_extraction_id === originalId` and
`extraction_id !== originalId`. Full readability ‚Äî hitting `GET
/extractions/:id` on the old id after retry ‚Äî is covered by DIS-104
(end-to-end retry flow).
