# DIS-133 — useShortcuts hook (Enter/Esc/N/P)

## Summary
Keyboard-shortcut hook for the Verify flow: `Enter` = approve, `Escape` = cancel, `N`/`P` = next/previous field. Pure `matchShortcut()` helper drives the React `useEffect`-based `useShortcuts()` hook, with modifier keys and focused text inputs correctly suppressed.

## Files
- `dis/ui/src/hooks/useShortcuts.ts` — `ShortcutHandlers`, `ShortcutAction`, pure `matchShortcut()`, `useShortcuts()` hook
- `dis/ui/tests/shortcuts.spec.ts` — Playwright-runner-compatible unit tests (pure logic, no browser)
- `dis/handoffs/DIS-133.md` — this file

## Decisions
- **Pure helper exported** — `matchShortcut(key, opts)` is a side-effect-free function taking a KeyboardEvent-like shape. Hook tests stay trivial and don't need jsdom or a real browser.
- **Input suppression by target tag** — `INPUT`/`TEXTAREA`/`SELECT` never fire shortcuts; callers can also pass `disabled: true` (e.g., when a modal dialog owns focus).
- **Modifier keys ignored** — any of Ctrl/Meta/Alt returns `null` so we don't clash with browser/OS shortcuts. (Shift is intentionally allowed so `N`/`P` capitals still match.)
- **Test runner = Playwright `test`** — per DIS-131 precedent: vitest not in devDependencies, so Playwright's test harness is reused for pure-logic specs (matches `tests/flags.spec.ts` pattern). The exact required `test('N advances focus to next field', …)` name is preserved.
- **No help-overlay component yet** — the brief mentions "Help overlay" but only lists the hook + test + handoff under "Files allowed (STRICT)". Overlay UI deferred to a follow-up that pairs it with the Verify page integration.
