## 1. What shipped

- `dis/tests/integration/error-envelope-e2e.test.ts` — 7 tests.

Composes `correlationId()` middleware (DIS-008) with the DIS-029 `toEnvelope()`
builder as a Hono `onError` handler. Drives six routes, each throwing one
known DIS error class in turn:

| Route                    | Thrown                             | Expected `error.code`        |
| ------------------------ | ---------------------------------- | ---------------------------- |
| `/invalid-state`         | `InvalidStateTransitionError`      | `INVALID_STATE_TRANSITION`   |
| `/version-conflict`      | `VersionConflictError` (core)      | `VERSION_CONFLICT`           |
| `/ocr-timeout`           | `OcrProviderTimeoutError`          | `OCR_PROVIDER_TIMEOUT`       |
| `/ocr-rate-limited`      | `OcrProviderRateLimitedError`      | `RATE_LIMITED`               |
| `/structuring-invalid`   | `StructuringSchemaInvalidError`    | `STRUCTURING_SCHEMA_INVALID` |
| `/generic`               | plain `Error`                      | `INTERNAL_ERROR`             |

Plus an extra test asserting a fresh UUIDv4 `correlation_id` is minted when
no inbound `x-correlation-id` header is present.

## 2. Shape asserted

Per `04_api/error_model.md`, the canonical minimal envelope the test asserts is:

```json
{
  "error": {
    "code": "<UPPER_SNAKE>",
    "message": "<non-empty>",
    "correlation_id": "<uuid-or-inbound>"
  }
}
```

This matches the shape returned by DIS-029's `toEnvelope()`. The richer
DIS-005 middleware envelope (which additionally carries `retryable` and
`request_id`) is a superset — this test pins the *minimum* contract
required by the brief.

## 3. Adaptation notes

- The Wave-2b brief references "DIS-005's error-envelope middleware" but
  the asserted shape `{code, message, correlation_id}` matches DIS-029's
  builder, not the DIS-005 middleware shape. I integrated the two: the
  test wires a Hono `onError` handler that delegates to `toEnvelope()`
  so the DIS-029 name-derivation + explicit overrides drive `code`.
- This still uses Hono + `app.fetch(request)` for a true HTTP-level
  round-trip; no network socket is bound (hermetic, parallel-safe).
