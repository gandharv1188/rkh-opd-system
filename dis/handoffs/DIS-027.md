## 3. Decisions taken during implementation

### D-1: UTF-8 for string input

**Context:** string input has no inherent byte encoding.
**Options considered:** UTF-8, ASCII, explicit-only (reject strings).
**Decision:** UTF-8 via `hash.update(str, 'utf8')`.
**Reason:** Matches idempotency key hashing and storage-integrity call sites (clinical documents are multibyte-capable). Aligns with `node:crypto` default.
**Revisit if:** a call site needs non-UTF-8 bytes — then it should pre-encode to `Buffer` itself.

## 4. What was deliberately NOT done

- Streaming API (`sha256Stream`) — no current caller needs it; add when DIS-053 storage upload lands if large blobs show up.
- HMAC variants — out of scope (different primitive, different keying story).
