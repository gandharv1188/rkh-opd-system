## Summary
React `ErrorBoundary` class component that catches render errors, renders a default fallback UI (with `role="alert"` and `data-testid="error-boundary-fallback"`), and fire-and-forgets an error report (message, stack, componentStack, correlation_id) to `/admin/errors` (override via `reportEndpoint` prop). Optional `fallback(error)` render override supported.

## Files
- `dis/ui/src/components/ErrorBoundary.tsx` — the component
- `dis/ui/tests/error-boundary.spec.ts` — Playwright test
- `dis/handoffs/DIS-128.md` — this file

## Design notes
- `static getDerivedStateFromError` sets `{ error }` — pure, side-effect free (correct React contract).
- `componentDidCatch` reads `window.__disCorrelationId` and POSTs JSON; failures swallowed so the boundary never re-throws during reporting.
- Fallback UI wraps the error message in `<details>` so technical info stays collapsed.
- Default export omitted per repo convention — consumers import the named class.

## Tests
- Playwright spec `tests/error-boundary.spec.ts`, `test.describe('ErrorBoundary')`, `test('renders fallback on thrown error', ...)`.
- Smoke: asserts app shell (`topbar`) still renders.
- Behaviour: invokes `getDerivedStateFromError` via `page.evaluate` as a pure static-method check and asserts the returned state carries the thrown error.
- Full `<ErrorBoundary>` render-time fallback UI assertion requires `@testing-library/react` (not yet installed in dis/ui). Filed as follow-up **DIS-128-followup** — add testing-library + a harness route that mounts a throwing child, then assert `getByTestId('error-boundary-fallback')` is visible and the fetch report body matches shape.
