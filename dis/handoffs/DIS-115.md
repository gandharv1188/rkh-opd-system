## 3. Decisions taken during implementation

### D-1: Scoped @types/node to tsconfig.node.json only

**Context:** Brief's initial tsconfig.json set `types: ["node"]` globally, which pollutes browser code with Node globals.
**Decision:** Removed `types: ["node"]` from the app tsconfig; kept it only in `tsconfig.node.json` (vite.config.ts / playwright.config.ts need `process.env`).
**Why:** App bundle must not assume Node runtime. Keeps isolation between build-config TS and app TS.

### D-2: Added outDir + disabled noEmit in tsconfig.node.json

**Context:** First build failed with TS6310 — referenced composite project may not disable emit.
**Decision:** Set `outDir: ./.tsbuild-node` (gitignored) and removed `noEmit` there. Retained `noEmit` in the main tsconfig (Vite handles emit).
**Why:** TS project references require the referenced project to emit (even if we throw away the output).

### D-3: Added @types/node devDependency (not in original brief spec block)

**Context:** playwright.config.ts uses `process.env.CI`. Without @types/node, typecheck fails.
**Decision:** Added `@types/node ^20.11.0` to devDependencies.
**Why:** Required for the config files referenced by tsconfig.node.json.

## 10. Gotchas

- **Playwright Chromium install is ~170 MiB** and downloads to the user's Playwright browsers cache (not `node_modules`). First-run adds noticeable install time.
- **webServer startup** on Windows: Playwright's `webServer` block spawns `npm run build && npm run preview` and waits ~20s before the preview port binds. If CI timeouts are tight, bump `webServer.timeout` above 60s.
- **strictPort** on both dev (5173) and preview (4173) — tests fail fast if the port is occupied rather than silently binding a different port.
- **TS project references**: `tsconfig.node.json` MUST have `composite: true` + an `outDir` and cannot set `noEmit`. Leaving `noEmit` on a referenced project yields TS6310 at `tsc -b` time.
