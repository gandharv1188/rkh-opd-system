# DIS-099 — GET /admin/metrics (Wave-5 test location)

## Summary
DIS-099 in the Wave-5 backlog requests the admin-metrics test at the
Wave-5-convention path `dis/tests/http/admin-metrics.test.ts`. The route
itself (`dis/src/http/routes/admin-metrics.ts`) and an earlier integration
test (`dis/tests/integration/admin-metrics.test.ts`) already existed from
DIS-009. This ticket adds a thin, parallel test at the new path without
touching production code or the DIS-009 test.

## Files touched

- `dis/handoffs/DIS-099.md` — **new** (this file)

No production edits. No changes to `dis/tests/integration/admin-metrics.test.ts`.

## Approach
`createServer` does **not** wire `registerAdminMetricsRoute` (verified in
`dis/src/http/server.ts`). Per the brief, the test composes a minimal app
directly:

```ts
const app = new Hono<{ Variables: AppVariables }>();
app.use('*', correlationId());
registerAdminMetricsRoute(app as unknown as Hono, metrics);
```

This mirrors the pattern used by every other Wave-5 route test in
`dis/tests/http/`, and keeps the test hermetic (no shared server state,
no network).

## Test cases
1. **returns 200 JSON snapshot with counters + gauges arrays** — seeds
   `docs_ingested_total` (unlabelled + `{provider:'chandra'}`), asserts
   status/content-type, and verifies both label variants round-trip.
2. **every counter value is a finite number** — translation of the
   original VERIFY-2 `curl | jq` check. The real snapshot shape has no
   `queue_depth` field; the equivalent invariant for this shape is that
   every `CounterSample.value` is `typeof 'number'` and `Number.isFinite`.
3. **empty metrics yields empty arrays** — baseline shape assertion on
   a fresh `createMetrics()` instance.

## Deviations from backlog VERIFY block
- **Original VERIFY-2** was `curl http://localhost:3000/admin/metrics | jq .queue_depth`. Skipped as documented: (a) requires a running server, out of scope for a unit-style test; (b) the snapshot has no `queue_depth` field — `MetricsSnapshot = { counters: CounterSample[]; gauges: CounterSample[] }`. The numeric-invariant assertion in test #2 is the direct equivalent over the real shape.
- No `correlation_id` echo assertion: the route body is a raw `metrics.snapshot()` pass-through; it does not embed `correlation_id`. Existing middleware already tested separately (`dis/tests/http/error-handler.test.ts` etc.).
