## 3. Decisions taken during implementation

### D-1: `rawResponse = wb` (reference preservation)

**Context:** Brief (and DatalabChandraAdapter precedent) called for byte-identical CS-2 preservation; the full `XLSX.WorkBook` contains live cell-object references that don't survive `JSON.stringify`.
**Choice:** Preserve the workbook object by reference. In-memory callers get faithful state; downstream JSON serialization will lossy-drop non-serializable pieces, which is acceptable for audit.
**Why:** Matches sibling adapter pattern; avoids premature serialization.

### D-2: Markdown tables with padded cells

**Context:** Sheets may have ragged rows (fewer cells than header). `sheet_to_json({header:1})` returns ragged arrays.
**Choice:** Iterate header length and coerce missing cells to `''`. Produces valid GFM tables.

### D-3: Strict-null guard on `wb.Sheets[name]`

**Context:** `tsc --strict` flagged possible `undefined` when indexing the `Sheets` record.
**Choice:** `if (!ws) continue` defensively, even though `SheetNames` entries are always present in practice.

### D-4: node_modules junction in worktree

**Context:** Running `npx vitest` from worktree `dis/` fails because Node's upward-walk resolver doesn't find main's `dis/node_modules` (walks up past the `dis` segment). Brief told agents not to `npm install`.
**Choice:** Created a Windows junction `dis/node_modules` → main `dis/node_modules` via PowerShell `New-Item -ItemType Junction` (cmd `mklink /J` had MSYS path-mangling issues in Git Bash). Junction is gitignored — not part of the commit.
**Follow-up:** Orchestrator may want to pre-create this junction during worktree setup.
