## 1. What was built

- `dis/src/adapters/secrets/supabase-secrets.ts` — `SupabaseSecretsAdapter` implementing `SecretsPort`.
  - In-memory `Map<string, { value, expiresAt }>` cache keyed by secret name; TTL defaults to 5 min, configurable via `ttlMs`.
  - On cache hit within TTL: returns cached value, no network call.
  - On miss: calls Supabase Vault's `decrypted_secrets` PostgREST view with service-role headers (`apikey` + `Authorization: Bearer`). PostgREST `eq.<name>` filter with `encodeURIComponent` keeps the URL parameterised.
  - Falls back to `process.env` (or an injectable `env` map for tests) when `supabaseUrl` or `serviceRoleKey` is not configured — enables Edge Function / local dev / test contexts without a Supabase project.
  - Exports `SecretNotFoundError` — thrown on empty vault result, HTTP 404, or missing env key. Callers treat this as terminal misconfiguration.
  - Non-404 HTTP failures bubble as `Error` and are **not** cached; retry on the next call works.
  - Injectable `fetch`, `now`, `env` seams.

No cross-adapter imports. No new npm deps.
