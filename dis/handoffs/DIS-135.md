## Public API

```ts
startVerifyTimer(extractionId, now?): void
stopVerifyTimer(extractionId, outcome, now?): VerifyDuration | null
postVerifyDuration(duration, endpoint?): Promise<void>
__resetTimers(): void  // test-only
```

`VerifyDuration` carries `extraction_id`, `started_at`, `ended_at`, `duration_ms`, `outcome` (`'approved' | 'rejected' | 'abandoned'`). Injectable `now: () => number` for deterministic testing. `postVerifyDuration` swallows fetch errors so telemetry never crashes the verify UI.

## Test coverage

- Duration math between start/stop with injected clock.
- Returns `null` when no timer started for the given id.
- Stop clears the timer (second stop for same id returns null).
- All three outcomes (`approved`, `rejected`, `abandoned`) round-trip.
