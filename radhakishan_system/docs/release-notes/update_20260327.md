## Summary

## 1. Universal Dose Calculator

**What changed:** The dose calculator has been rebuilt from the ground up.

- **Combo drugs now work correctly.** For drugs like Wikoryl AF (Phenylephrine + Chlorpheniramine), the calculator shows the dose for EACH ingredient separately, checks max doses for each, and identifies the limiting ingredient.
- **Universal slider for all drugs.** Previously, "fixed-dose" drugs (like drops) had a simple +/- stepper. Now ALL drugs use the same slider interface — whether weight-based, fixed-dose, BSA-based, or age-based.
- **Route selection.** When a drug has multiple routes (e.g., Paracetamol oral vs rectal), route radio buttons appear so the doctor can switch and see route-specific dosing.
- **Custom concentration.** An "Edit" button next to the strength radios allows entering custom concentrations for formulations not in the database.

**Bug fixed:** When adding a medicine manually from the formulary, the dose was showing in mg instead of mL (e.g., Cefixime showed "78.7 mL" instead of "3.9 mL"). The mg-to-mL concentration conversion was missing. Now fixed.

**Pictogram fix:** The medication pictogram icons (subah/dopahar/sham) now correctly match the prescribed frequency. Previously, all medicines showed "subah-sham" (BD) regardless of actual frequency. Now: OD = morning only, BD = morning+evening, TDS = morning+afternoon+evening.

## 2. Formulary Database Enrichment

**680 drugs** across 3 data files have been upgraded:

- **Per-ingredient dosing bands.** Every dosing band now has an `ingredient_doses[]` array with dose ranges, max doses, and SNOMED codes for EACH ingredient. For combo drugs, each ingredient has its own safe range from authoritative sources.
- **48 combo drugs fully enriched** with per-ingredient data from 5 parallel research agents using Harriet Lane Handbook, BNFC, Nelson's Textbook, WHO guidelines, FDA labels, NIH ARV Guidelines, and AAP Red Book.
- **18 emergency drugs** (Adrenaline, Midazolam, Ketamine, Dopamine, etc.) received SNOMED CT substance codes.
- **Zero remaining gaps** — all secondary ingredients classified (filled with dose data, or marked as excipient/vehicle/data error).

**Data quality:** 93% overall quality score. Identified 3 drugs with incorrectly merged ingredients (data import errors flagged for cleanup).

## 4. AI Clinical Summary + Prescription Generation

- **Comprehensive clinical summary.** The `generate-visit-summary` Edge Function now produces a single comprehensive summary covering BOTH prior visit history AND uploaded document findings (lab reports, discharge summaries, imaging scans). Previously, only prior prescriptions were summarized; uploaded documents were ignored.
- **First-time patients with documents now get a summary.** Previously, only returning patients received an AI summary. Now, even first-time patients who bring lab reports or discharge summaries get a clinical summary from the document OCR findings.
- **Summary generated after registration completes.** The summary runs as a background task after the receptionist sees the success ticket — it does NOT block registration or token printing. The summary is saved to the visit as soon as Claude responds.
- **Document OCR summaries included.** Reception uploads scanned documents during registration. The OCR text is extracted, then passed to the summary generator so the doctor sees a consolidated view of prior history + current document findings.
- **Visit summary sent to AI for prescription.** The comprehensive summary (which now includes document findings) flows to Claude when generating the prescription (was broken due to a code typo — `._visit` instead of `.visit`, now fixed).
- **Category and therapeutic use** now included in formulary data sent to Claude (helps AI match drugs to diagnoses).
- **Token optimization.** Empty fields stripped from AI context (saves ~10% tokens per drug lookup). Ingredient SNOMED codes removed from AI context (not needed for dosing).
- **Standard Rx enriched fields** (expected_course, key_clinical_points, severity_assessment, monitoring_parameters) now sent to Claude when it looks up a standard protocol.
